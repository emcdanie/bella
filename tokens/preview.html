Tier 3

Components

Every component reference is resolved through semantic, so flipping dark mode flips the components automatically.

Button

Primary call-to-action. Warm amber background for primary, ink text across variants. Reserved for the single most important action in a view.

Variants: primary · secondary · ghost

States: default · hover · active · focus · disabled

Use for: Hero CTA, Card 'Read more' action, Form submit

Don't use for: Do not use more than one primary button per view; Do not stack two buttons of the same variant

Card

Editorial glass-first card system. Three primary tiers progress from subtle panel (default) to hero glass (glass). Plus featured (solid bright) and inverse (dark-mode embed).

Variants: default · elevated · glass · featured · inverse

States: resting · hover · focus

Use for: Portfolio grid, CHIP idea clusters, Dashboard tiles, Case-study cards, Hero feature panels

Don't use for: Do not nest cards inside cards — glass-on-glass becomes mud; Do not use for navigation — that's nav-link's job; Do not place glass over amber/steel/dusk/sage — the contrast model breaks

Tag

Uppercase mono label. Categorizes a card, a project, a mood. Ink-on-amber-wash is the canonical AAA-safe look.

Variants: accent · info · muted

States: default

Use for: Card category labels, Section eyebrows inside content areas, Filter state chips

Don't use for: Do not set foreground to amber/steel/dusk/sage on parchment — those fail AAA; Do not place on amber backgrounds without verifying the foreground contrast

Eyebrow

Small mono label above a heading. Signals context or category.

Variants: accent · muted

States: default

Use for: Above hero headlines, Above section titles, Above card groupings

Don't use for: Do not use as a heading — eyebrows complement a heading, never replace one; Do not hard-code amber as the foreground — flow through text-accent so AAA enforcement stays centralized

Input

Text input. Serif-on-surface, linen border, amber focus ring.

Variants: default · search

States: default · hover · focus · disabled · error

Use for: Forms, Search bars, Filter inputs

Don't use for: Do not strip the border on default state — it's how inputs are recognizable

Link

Inline text link inside body copy.

Variants: default

States: default · hover · visited · focus

Use for: Inline references inside paragraphs, Footer links

Don't use for: Do not use for primary navigation — that's nav-link; Do not rely on color alone — always underlined or weight-shifted on hover

Nav Link

Top-level navigation item. Mono, uppercase-lite. Active state reads as ink text with a 2px amber underline — amber is decoration, ink carries the letters.

Variants: default · active

States: default · hover · active · focus

Use for: Top navigation bar, Footer navigation

Don't use for: Do not mix nav-link and inline-link in the same cluster; Do not color-distinguish the active state with amber text — use the amber underline token instead

Section

A vertical slice of a page. Defines standard padding and gap for content blocks.

Use for: Case-study sections, Portfolio page regions

Don't use for: Do not nest sections inside sections

Modal

Modal dialog scaffold. Backdrop blur over the page content, centered dialog surface above.

Variants: default

States: open · closed

Use for: Dialogs, Sheets, Confirmations

Don't use for: Do not nest modals; Do not use for non-blocking UI — use a drawer or popover instead

Shadows & Glass

Three glass tiers, four shadow specimens, five blur steps. Warmth comes from parchment showing through translucent layers — white alpha is an overlay, never a solid fill.

Primary. Hero panel, feature card. 18px blur, 24px radius, shadow.raised with a crisp inset glass-90 highlight.

card.elevated

Mid-tier. Case-study cards, content panels. 16px blur, 20px radius, shadow.card-elevated with a glass-68 inset.

card.default

Subtle panel. Secondary surface, nested container. 14px blur, 16px radius, shadow.card-default with a glass-55 inset.
